제이쿼리 문서객체모델_객체탐색(traversing)

문서객체모델(DOM) : 제이쿼리 traversing(탐색)메서드는 기존의 선택자보다 좀더 html문서의 요소들을 정확하게 선택해 주는 메소드이다.

html문서의 구조에서 부모요소를 찾거나 부모의 형제들, 부모아래 자식요소들, 자식요소에서 부모의 부모들을 찾을 수 있는 메소드들이다.

1. 문서안에서 요소들 중 부모요소 찾기

  • parent() 선택한 요소의 바로 위에 위치한 부모요소를 선택
  • parents() 선택한 요소의 위에 위치한 모든 부모, 조상 요소들을 모두 선택
  • parentUntil() 선택한 요소에서부터 지정한 요소까지 부모요소를 선택

2. 문서안에서 요소들 중 자식요소 찾기

  • children() 선택한 부모요소의 아래 자식요소를 선택
  • find('이름') 선택한 부모요소의 자식요소들 중에서 특정 이름을 가진 자식요소만 선택

3. 문서안에서 요소들 중 형제요소(sibling) 찾기

Siblings메소드 : 사용자가 선택한 요소를 기준으로 형제관계에 있는 (동일한)요소에서 이전, 다음 요소들을 선택할 때 사용

  • siblings() 형제관계 요소들 중 자신을 제외한 나머지 형제들을 선택
  • next() 현재요소의 다음 요소를 선택
  • nextAll() 현재요소의 다음에 있는 모든 요소를 선택
  • nextUntil() 현재요소의 다음에서부터 ~ 이전까지 선택
  • prev() 현재요소의 이전에 있는 요소를 선택
  • prevAll() 현재요소의 이전에 있는 모든 요소를 선택
  • prevUntil() 현재요소의 이전 요소부터 ~ 전까지를 선택

4. 기타 탐색(traversing) - 필터링 메서드

html문서의 구조에서 사용자가 선택한 요소를 정확하게 찾아낼 수 있는 메소드 중의 하나이다.

  • first() 형제관계 요소들 중에서 첫번째 요소를 선택
  • last() 형제관계 요소들 중에서 마지막번째 요소를 선택
  • eq(n) 인덱스값의 n번째에 해당하는 요소를 선택
  • filter() 사용자가 지정한 이름에 맞는 요소를 선택
  • not() 사용자가 지정한 이름이 아닌 요소를 선택
